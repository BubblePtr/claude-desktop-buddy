# 移植规格：M5StickC Plus → ESP32-S3-Touch-AMOLED-1.8

## 目标

将固件从 M5StickC Plus 直接移植到 Waveshare ESP32-S3-Touch-AMOLED-1.8，不维护双版本。所有 M5 特定代码就地替换。

当 `README.md`、代码和 `platformio.ini` 冲突时，以代码和 `platformio.ini` 为准；文档需显式标注并修正这些不一致。

---

## 硬件对比

| 组件 | M5StickC Plus（当前）| ESP32-S3-AMOLED-1.8（目标）|
|---|---|---|
| 屏幕 | 135×240 RGB565 IPS | 368×448 AMOLED |
| 显示驱动 | TFT_eSPI | SH8601 (QSPI) |
| 渲染库 | TFT_eSPI + TFT_eSprite | Arduino_GFX + Arduino_Canvas |
| IMU | MPU6886 | QMI8658 |
| 电源 IC | AXP192 | AXP2101 |
| RTC | M5.Rtc (内置) | PCF85063 (I2C) |
| 按键 A | BtnA (GPIO37) | BOOT (GPIO0, 低有效) |
| 按键 B | BtnB (GPIO39) | AXP2101 短按事件 |
| 触摸屏 | 无 | FT3168 (I2C，中断 GPIO21) |
| 蜂鸣器 | M5.Beep | 无（删除） |
| LED | GPIO10 红灯（低有效）| 无（改为屏幕边框动画）|
| GPIO 扩展器 | 无 | TCA9554 (I2C) |
| MCU | ESP32 LX6 | ESP32-S3 LX7 |
| PSRAM | 无 | 8MB OPI |

## 关键设计决策

### 1. 逻辑画布：184×224（半分辨率）

物理分辨率 368×448，但渲染逻辑画布取一半：**W=184，H=224**，推屏时 2× 最近邻放大。

理由：
- 保留原版的"像素感"外观，与 ASCII 角色风格一致
- 坐标系与原版（135×240）接近，改动量小
- PSRAM 占用低（约 80KB canvas）

推屏函数在 `hwDisplayPush()` 中逐行做 2× upscale，主循环中的 `spr.pushSprite(0, 0)` 替换为此函数调用。

### 2. 横屏时钟：简化为仅竖屏

原版 landscape clock 通过 `M5.Lcd.setRotation()` 切换硬件旋转直接绘制。新板 AMOLED 不支持运行时硬件旋转（Canvas 方案需重建画布）。

决策：**删除横屏分支**，只保留竖屏时钟。新屏高宽比 (224/184 ≈ 1.22) 比原版 (240/135 ≈ 1.78) 更方，纵向空间反而更充裕，竖屏时钟显示效果更好。

影响：删除 `clockOrient`、`clockUpdateOrient()`、`landscapeClock` 变量，以及 `buddyRenderTo()`、`characterRenderTo()` 对 `M5.Lcd` 的直接绘制路径。

### 3. BLE：NimBLE 替换 ArduinoBLE

原版用 ESP32 Arduino 内置的 Bluedroid 栈（`BLEDevice::init()` 等）。ESP32-S3 上推荐用 NimBLE（`h2zero/NimBLE-Arduino`），内存占用更小、更稳定。

`ble_bridge.cpp` 需重写，但对外接口（`bleInit/bleConnected/bleSecure/blePasskey/bleRead/bleWrite`）保持不变，`main.cpp` 无感知。

NimBLE 与原版的主要差异：
- 初始化：`NimBLEDevice::init()` 代替 `BLEDevice::init()`
- 安全回调：`NimBLESecurityCallbacks` 接口不同（`onPassKeyNotify` → `onPassKeyDisplay`）
- Bond 管理：`NimBLEDevice::deleteBond()` 替代 `esp_ble_remove_bond_device()`
- MTU 回调：通过 `NimBLEServerCallbacks::onMTUChange()` 获取

### 4. 按键映射

| 行为 | 原版 | 新版 |
|---|---|---|
| 按键 A（短按释放）| BtnA.wasReleased() | BOOT(GPIO0) 上升沿 |
| 按键 A（长按 600ms）| BtnA.pressedFor(600) | BOOT 持续低电平 600ms |
| 按键 B（短按）| BtnB.wasPressed() | AXP2101 PKEY 短按事件 |
| 屏幕 on/off | AXP.GetBtnPress()==0x02 | AXP2101 PKEY 短按事件（重用为 B）|
| 关机 | AXP 长按 6s（硬件）| AXP2101 PKEY 长按事件 |
| 触摸屏 | 无 | 暂不映射到应用逻辑 |

**注意**：原版电源按键（AXP 短按）触发屏幕 on/off，现在 AXP2101 短按既是 BtnB 又可触发屏幕 off。冲突解决：在非 prompt 状态下，AXP 短按同时执行 BtnB 逻辑 + 屏幕 off toggle；在 prompt 状态下仅执行 deny（BtnB 逻辑）。

### 5. LED → 屏幕边框动画

原版 GPIO10 红灯在 `P_ATTENTION` 状态下 400ms 闪烁。新版无 GPIO LED，替换为：在每帧渲染后，若 `activeState == P_ATTENTION && settings().led`，在 canvas 顶部绘制一条 8px 高的红色（HOT）横条，否则绘制 `p.bg` 颜色横条（擦除）。

### 6. 蜂鸣器：删除

`M5.Beep.begin()`、`M5.Beep.update()`、`M5.Beep.tone()` 全部删除。`beep()` 函数改为空操作（保留调用点，方便未来接扬声器）：

```cpp
static void beep(uint16_t, uint16_t) {}
```

### 7. 坐标更新

| 常量 | 原值 | 新值 | 说明 |
|---|---|---|---|
| `W` | 135 | 184 | 逻辑画布宽 |
| `H` | 240 | 224 | 逻辑画布高 |
| `CX` | 67 | 92 | W/2 |
| `CY_BASE` | 120 | 112 | H/2 |
| `BUDDY_X_CENTER` | 67 | 92 | buddy 水平中心 |
| `BUDDY_CANVAS_W` | 135 | 184 | buddy 清除宽度 |
| `BUDDY_Y_BASE` | 30 | 30 | 保持（绝对像素感合适）|
| `PEEK_TOP` | 70 | 70 | 保持（info panel 分界线）|
| 菜单宽 `mw` | 118 | 165 | ~90% of W |
| 菜单行高 | 14 | 14 | 保持 |
| Approval AREA | 78 | 80 | 底部区域 |
| HUD WIDTH | 21 chars | 28 chars | W/(6px/char) |
| HUD SHOW | 3 lines | 3 lines | 保持 |

### `src/main.cpp`

| 原代码 | 替换为 |
|---|---|
| `#include <M5StickCPlus.h>` | `#include <Arduino_GFX_Library.h>`，以及各硬件头文件 |
| `TFT_eSprite spr = TFT_eSprite(&M5.Lcd)` | `Arduino_Canvas* spr` + `Arduino_SH8601* gfx` |
| `M5.begin()` | 各硬件独立初始化（按顺序：I2C → TCA9554 → 屏幕 → AXP2101 → 触摸 → IMU → RTC）|
| `M5.Lcd.setRotation(0)` | 删除 |
| `M5.Imu.Init()` | `qmi.begin(Wire, ...)` + `qmi.configAccelerometer(ACC_RANGE_2G, ACC_ODR_125Hz)` |
| `M5.Beep.begin()` | 删除 |
| `M5.update()` | 手动轮询 BOOT 按键 + AXP2101 IRQ |
| `M5.Beep.update()` | 删除 |
| `M5.Axp.ScreenBreath(n)` | `gfx->setBrightness(map(n, 20, 100, 30, 200))` |
| `M5.Axp.SetLDO2(false/true)` | `gfx->setBrightness(0)` / `gfx->setBrightness(brightness)` |
| `M5.Axp.GetBtnPress()` | AXP2101 IRQ 轮询 |
| `M5.Axp.GetBatVoltage()` | `axp.getBattVoltage() / 1000.0f` |
| `M5.Axp.GetBatCurrent()` | `0`（AXP2101 不暴露电流，`charging` 用 `axp.isCharging()` 替代）|
| `M5.Axp.GetVBusVoltage()` | `axp.getVbusVoltage() / 1000.0f` |
| `M5.Axp.GetTempInAXP192()` | `axp.getTemperature()` |
| `M5.Axp.PowerOff()` | `axp.shutdown()` 或软件复位 |
| `M5.Imu.getAccelData(&ax, &ay, &az)` | `qmi.getAccelerometer(ax, ay, az); az = -az` （Z 轴翻转）|
| `M5.Rtc.GetTime(&t)` | `pcf.getDateTime()` 后解包 |
| `M5.Rtc.GetDate(&d)` | 同上 |
| `M5.Rtc.SetTime(&t)` | `pcf.setDateTime(...)` |
| `M5.BtnA.isPressed()` | `digitalRead(PIN_BOOT) == LOW` |
| `M5.BtnA.pressedFor(600)` | 手动计时 |
| `M5.BtnA.wasReleased()` | 上升沿检测 |
| `M5.BtnB.wasPressed()` | AXP2101 短按事件 |
| `spr.pushSprite(0, 0)` | `hwDisplayPush(spr)` （2× upscale 推屏）|
| `M5.Lcd.setRotation(n)` / `M5.Lcd.fillScreen()` 等横屏调用 | **删除**（横屏时钟功能移除）|
| `digitalWrite(LED_PIN, ...)` | 边框动画（见上文）|
| `GREEN`（TFT_eSPI 内建色）| `0x07E0`（RGB565 绿色）|
| `RED`（TFT_eSPI 内建色）| `0xF800` |

### `src/ble_bridge.cpp`

完整重写，接口不变。NimBLE 关键实现点：

```cpp
// 初始化
NimBLEDevice::init(deviceName);
NimBLEDevice::setMTU(517);
// 安全
NimBLEDevice::setSecurityAuth(BLE_SM_PAIR_AUTHREQ_SC | BLE_SM_PAIR_AUTHREQ_MITM | BLE_SM_PAIR_AUTHREQ_BOND);
NimBLEDevice::setSecurityIOCap(BLE_HS_IO_DISPLAY_ONLY);
// 安全回调（NimBLESecurityCallbacks 子类）
uint32_t onPassKeyRequest() → 返回 0（DisplayOnly 不需要）
void onAuthenticationComplete(ble_gap_conn_desc*) → 更新 secure/passkey
// Bond 清除
NimBLEDevice::deleteAllBonds();
```

`blePasskey` 通过 `NimBLEDevice::getSecurityCallbacks()` 对应的回调中 `NimBLEDevice::startSecurity(connHandle)` 流程获取，实现方式与原版略有差异，但 `blePasskey()` 对外签名不变。

### `src/buddies/*.cpp`（18 个物种文件）

这些文件的动画逻辑本身无需改动；它们主要通过 `buddyPrintLine`、`buddyPrintSprite`、`buddySetCursor`、`buddySetColor`、`buddyPrint` 间接使用绘图 API。

但移植时仍需完成类型/包含替换：例如将 `#include <M5StickCPlus.h>` 替换为 `#include <Arduino_GFX_Library.h>`，将 `extern TFT_eSprite spr` 改为 `extern Arduino_Canvas* spr`，以及把任何直接绘图调用从 `spr.xxx()` 机械替换为 `spr->xxx()`。

## 待确认事项

1. **`no_ota_8mb.csv`**：确认 LittleFS 分区大小足够存储 GIF 角色（约 700KB），与原版一致。
2. **PCF85063 时区**：PCF85063 存储 UTC 还是本地时间取决于 Claude 桌面端如何推送。行为应与原版一致（桌面端通过 BLE 下发本地时间写入 RTC）。
3. **触摸屏**：本次移植中触摸屏**不映射到任何应用逻辑**，仅初始化以避免 I2C 地址冲突报错。未来可扩展为滑动翻页等手势。
4. **Settings `clockRot`**：移除后，NVS 里已有此字段的设备 factory reset 后会消失，无兼容性问题。
5. **`buddies/*.cpp` grep**：执行 `grep -r "spr\." src/buddies/` 确认是否有直接绘图调用。
